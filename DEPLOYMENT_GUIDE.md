# خطوات جعل تطبيق Wood & More جاهزاً للتجربة الحقيقية

## نظرة عامة
التطبيق يعمل حالياً بتخزين محلي (قاعدة بيانات SQLite على الموبايل، و SharedPreferences على الويب). لا يوجد سيرفر خلفي، لذا كل البيانات على جهاز المستخدم أو متصفحه.

---

## المرحلة 1: التحضير قبل البناء

### 1.1 إزالة شريط التصحيح (Debug Banner)
- في وضع **Release** يختفي الشريط الأصفر تلقائياً.
- إذا ظهر أثناء التطوير، يمكن إزالته من `main.dart`:
```dart
MaterialApp(
  debugShowCheckedModeBanner: false,  // أضف هذا السطر
  ...
)
```

### 1.2 التأكد من الأصول (Assets)
- تأكد وجود ملف الشعار: `assets/images/logo.png`
- إذا أضفت خطوطاً عربية للـ PDF تأكد تسجيلها في `pubspec.yaml` تحت `flutter.fonts`

### 1.3 تحديث رقم الإصدار (اختياري)
في `pubspec.yaml`:
```yaml
version: 1.0.0+1   # 1.0.0 = versionName ، 1 = versionCode (Android)
```
غيّرها عند كل إصدار جديد (مثلاً 1.0.1+2).

## المرحلة 2: البناء للتجربة على أجهزة حقيقية

### 2.1 بناء نسخة الويب (للتجربة عبر المتصفح)

```bash
cd D:\Helal\FlutterProjects\wood_and_more_app
flutter pub get
flutter build web
```

- الناتج يكون في مجلد: `build/web/`
- يمكنك رفع محتويات هذا المجلد إلى أي استضافة ويب (مثل: Firebase Hosting، Netlify، أو خادمك).
- **للتجربة المحلية بدون إنترنت:** شغّل `flutter run -d chrome` ثم من القائمة اختر "Install Wood & More app" (إن وُجد) أو شارك الرابط المحلي مع المختبرين على نفس الشبكة.

**نشر سريع على Firebase Hosting (مثال):**
```bash
firebase init hosting
# اختر build/web كمجلد عام
firebase deploy
```

### 2.2 بناء تطبيق أندرويد (APK للتجربة المباشرة)

```bash
cd D:\Helal\FlutterProjects\wood_and_more_app
flutter pub get
flutter build apk --release
```

- ملف الـ APK الناتج: `build/app/outputs/flutter-apk/app-release.apk`
- انسخ هذا الملف وشاركه مع المختبرين؛ يمكنهم تثبيته على هواتف أندرويد بعد السماح بـ "التثبيت من مصادر غير معروفة".

**لأحجام أصغر (حسب نوع المعالج):**
```bash
flutter build apk --split-per-abi --release
```
يعطيك عدة APK (armeabi-v7a، arm64-v8a، x86_64)؛ وزّع المناسب لجهاز كل مستخدم.

### 2.3 بناء App Bundle لأندرويد (لرفع متجر Google لاحقاً)

```bash
flutter build appbundle --release
```

- الناتج: `build/app/outputs/bundle/release/app-release.aab`
- يُستخدم عند رفع التطبيق إلى Google Play (وليس للتثبيت المباشر من الملف).

### 2.4 بناء آيفون / iPad (يتطلب جهاز Mac و Xcode)

```bash
flutter build ios --release
```
ثم فتح المشروع في Xcode وإعداد الشهادات والـ Provisioning Profile ورفع التطبيق أو توزيعه عبر TestFlight.

## المرحلة 3: الإعدادات الخاصة بكل منصة

### 3.1 أندرويد
- **اسم التطبيق:** عدّل `android/app/src/main/AndroidManifest.xml` (label) و/أو `android/app/build.gradle` (applicationId).
- **الأذونات:** التطبيق يستخدم الموقع (الحضور والانصراف) وملفات (المرفقات). تأكد أن الأذونات مذكورة في `AndroidManifest.xml` ولا تطلب أذونات زائدة.
- **التوقيع (للتوزيع خارج المتجر):** لبناء APK موقع للتجربة:
  - أنشئ keystore مرة واحدة ثم أضف في `android/key.properties` و `android/app/build.gradle` إعدادات التوقيع (يمكن اتباع الدليل الرسمي لـ Flutter: signing the app).

### 3.2 الويب
- **النطاق (Domain):** عند النشر على استضافة، استخدم HTTPS لتفعيل بعض APIs (مثل الموقع إن لزم).
- **حد التخزين:** ذكّر المختبرين أن المرفقات الكبيرة قد تُقلَّص أو لا تُحفظ بسبب حد تخزين المتصفح (تم معالجة الخطأ في الكود بتقليص تلقائي عند الحاجة).

## المرحلة 4: التجربة الفعلية من المستخدمين

### 4.1 قائمة تحقق قبل التوزيع
- [ ] تسجيل الدخول يعمل (البريد وكلمة المرور إن وُجدت، أو الطريقة الحالية).
- [ ] التقرير اليومي: إدخال البيانات، المرفقات (صور/مستند)، الحفظ بدون خطأ QuotaExceeded.
- [ ] تصدير PDF من شاشة التقارير (مع الخط العربي إن أضفته).
- [ ] واجهة المشروعات للمهندس: اختيار مشروع → منطقة → مبنى → تشوينات/قطعيات/نماذج.
- [ ] واجهة المالية (مدير المهندسين/مسؤول التطبيق): أرصدة، عهدة، تقارير مصروفات.
- [ ] الحضور والانصراف (إن كان يعتمد على الموقع: اختبار على جهاز حقيقي).

### 4.2 توزيع نسخة الويب
1. ارفع محتويات `build/web/` إلى الاستضافة.
2. شارك الرابط مع المختبرين (مثال: `https://your-domain.com/`).
3. يمكنهم فتح الرابط من أي متصفح (يفضّل Chrome أو Edge).

### 4.3 توزيع نسخة أندرويد
1. انشر ملف `app-release.apk` (عبر رابط تحميل، أو Google Drive، أو بريد).
2. اطلب من المختبرين تفعيل "السماح من هذا المصدر" لتثبيت التطبيقات من خارج المتجر.
3. بعد التثبيت، يفتحون التطبيق ويسجلون الدخول ويجرون السيناريوهات المطلوبة.

### 4.4 جمع الملاحظات
- حدد قناة واحدة لاستقبال الأخطاء والاقتراحات (مثل مجموعة واتساب أو نموذج).
- إذا ظهرت أخطاء (مثلاً على نوع معين من الأجهزة أو متصفح معين)، سجّل نوع الجهاز وإصدار النظام وخطوات إعادة الخطأ.

## المرحلة 5: بعد التجربة (تحسينات اختيارية)

- **نسخ احتياطي للبيانات:** حالياً لا يوجد سحابة؛ إذا أردت نسخاً احتياطية، ستحتاج لربط التطبيق بخادم أو خدمة تخزين سحابية لاحقاً.
- **تحديث الإصدار:** غيّر `version` في `pubspec.yaml` ثم أعد البناء والتوزيع.
- **متجر التطبيقات:** للنشر الرسمي على Google Play أو App Store تحتاج حساب مطور وإكمال إعدادات التوقيع والخصوصية ووصف التطبيق.
