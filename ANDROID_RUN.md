# تشغيل التطبيق على Android عبر USB

## تثبيت نسخة Release (APK) بشكل دائم

بعد تنفيذ `flutter build apk --release` يكون ملف التثبيت جاهزاً في:

```
build\app\outputs\flutter-apk\app-release.apk
```

**نقل الملف إلى الموبايل:**
1. وصّل الموبايل بالكمبيوتر بـ USB واختر **نقل ملفات (MTP)**.
2. انسخ الملف `app-release.apk` إلى مجلد في الموبايل (مثلاً **Download** أو **التنزيلات**).
3. افصل الكابل.

**التثبيت على الموبايل:**
1. افتح **مدير الملفات** وانتقل إلى المجلد الذي نسخت فيه الـ APK.
2. اضغط على `app-release.apk`.
3. إن ظهرت رسالة "تثبيت تطبيقات غير معروفة": اسمح للملفات أو لمتصفح/مدير الملفات بتثبيت التطبيقات من الإعدادات ثم أعد المحاولة.
4. اضغط **تثبيت** ثم **فتح**. سيظهر التطبيق **Wood & More** في درج التطبيقات مثل أي تطبيق آخر.

(يمكنك أيضاً إعادة تسمية الملف قبل النسخ، مثلاً `WoodAndMore.apk`.)

---

## متطلبات (للتشغيل عبر USB للتصحيح)
- هاتف Android موصول بالكمبيوتر **بكابل USB**.
- نفس شبكة الواي فاي (اختياري للـ USB؛ مفيد لاحقاً للتصحيح اللاسلكي).

---

## الخطوة ١: تفعيل وضع المطوّر و USB debugging

1. افتح **الإعدادات** على الهاتف.
2. اذهب إلى **حول الهاتف** (About phone).
3. اضغط على **رقم البناء** (Build number) **٧ مرات** حتى تظهر رسالة "أصبحت مطوّراً".
4. ارجع إلى الإعدادات وادخل **خيارات المطوّر** (Developer options).
5. فعّل **تصحيح أخطاء USB** (USB debugging).

---

## الخطوة ٢: توصيل الهاتف والسماح للكمبيوتر

1. وصّل الهاتف بالكمبيوتر **بكابل USB** (يفضّل كابل نقل بيانات وليس شحن فقط).
2. على الهاتف: اختر وضع USB **نقل ملفات (MTP)** أو **PTP** من إشعار USB (وليس "شحن فقط").
3. عند ظهور نافذة **"السماح بتصحيح أخطاء USB؟"** على الهاتف:
   - ضع علامة على **"السماح دائماً من هذا الكمبيوتر"** (إن وُجدت).
   - اضغط **موافق / السماح**.

---

## الخطوة ٣: التحقق وتشغيل التطبيق

في الطرفية (من مجلد المشروع):

```bash
# التحقق من ظهور الجهاز
flutter devices
```

يجب أن يظهر جهاز Android (مثل: `samsung ... • 1234567890ABCDEF • android-arm64`).

ثم شغّل التطبيق على الجهاز:

```bash
# تشغيل على أول جهاز Android يظهر
flutter run -d android
```

أو انسخ معرف الجهاز من نتيجة `flutter devices` واستخدمه:

```bash
flutter run -d <device-id>
```

مثال إذا كان المُعرّف `emulator-5554`:

```bash
flutter run -d emulator-5554
```

---

## إن لم يظهر الجهاز

- جرّب **منفذ USB آخر** في الكمبيوتر (يفضّل منفذ خلفي أو USB 2.0).
- على الهاتف: أوقف **تصحيح أخطاء USB** ثم فعّله مرة أخرى، ثم أعد توصيل الكابل.
- على ويندوز: إن طلب نظام التشغيل **تثبيت برنامج تشغيل** عند توصيل الهاتف، ثبّته أو ثبّت برنامج الهاتف الرسمي (Samsung / Xiaomi / etc.) لتحميل التعريفات.
- تأكد أن **USB debugging** لا يزال مفعّلاً بعد إعادة التشغيل.
