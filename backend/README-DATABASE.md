# Wood & More – قاعدة البيانات (PostgreSQL)

## استخدام قاعدة بيانات منفصلة للتطبيق

يجب تشغيل الجداول والبيانات الأولية داخل **قاعدة بيانات مخصّصة** للتطبيق فقط (مثلاً `wood_more`)، وليس داخل القاعدة الافتراضية أو الـ schema العام (public) لمُثيل PostgreSQL.

### الخطوات في Beekeeper (أو أي عميل SQL)

1. **إنشاء قاعدة البيانات (مرة واحدة)**  
   - اتصل بقاعدة البيانات الافتراضية (عادةً `postgres`).  
   - نفّذ الملف: **`01-create-database.sql`**  
   - النتيجة: إنشاء قاعدة بيانات باسم `wood_more`.

2. **الاتصال بقاعدة التطبيق**  
   - في Beekeeper: أنشئ اتصالاً جديداً (New Connection).  
   - اختر نفس المضيف والمنفذ والمستخدم، وغيّر **Database** إلى: **`wood_more`**.  
   - احفظ الاتصال واتصل به.

3. **تشغيل الجداول والبيانات الأولية**  
   - بعد الاتصال بقاعدة `wood_more`، نفّذ الملف: **`init-db.sql`**  
   - بذلك تُنشأ كل الجداول والبيانات الأولية داخل قاعدة `wood_more` فقط.

### إعداد الاتصال من التطبيق

في `config.json` (أو مكان إعداد الـ API) تأكد أن `apiBaseUrl` يشير إلى الخادم الذي يتصل بقاعدة **`wood_more`** (نفس اسم قاعدة البيانات في خطوة 1).

### ملخص الملفات

| الملف | متى يُنفَّذ | الاتصال المطلوب |
|--------|-------------|------------------|
| `01-create-database.sql` | مرة واحدة | الاتصال الافتراضي (مثلاً `postgres`) |
| `init-db.sql` | بعد إنشاء القاعدة | الاتصال بقاعدة `wood_more` |

بعد تنفيذ الخطوتين، تكون كل بيانات التطبيق معزولة داخل قاعدة البيانات `wood_more` فقط.
