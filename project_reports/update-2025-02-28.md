## Summary of Updates

This report summarizes changes detected from the working tree (uncommitted modifications and new files) and recent development activity. The project is a Flutter application for Wood & More with features for site engineers, project stores (inventory), daily reports, attendance, and admin dashboards.

**Highlights:**
- **Reports (PDF):** Daily and attendance reports now export with proper Arabic RTL and Tajawal font; daily report PDF redesigned to match the salary deduction form style and includes engineer check-in/check-out times.
- **Project stores & ledger:** Stock ledger added for project materials; automatic deduction from project stock when engineers save daily reports; “View ledger” per material with date and user name.
- **UI/UX:** Materials step in daily report fixed for mobile (overflow); material name and unit fields converted to dropdowns in project stores and staging (building materials) screens.
- **Backend:** New `project_stock_ledger` storage (SQLite + web), `getAttendanceForUserOnDate`, and deduction logic in `addDailyReport`.

## Feature / UI Changes

1. **التقارير اليومية (Daily reports)**  
   - PDF: Formal layout similar to salary deduction form (logo, green header, sections, tables).  
   - Shows engineer name, **check-in time**, **check-out time**, date, project, work details, materials table, expenses table.  
   - Arabic RTL and Tajawal font throughout.

2. **تقرير الحضور والانصراف (Attendance report PDF)**  
   - Arabic RTL and Tajawal font; table and header use `Directionality` and `textDirection: rtl`.

3. **مخازن المشاريع (الأرصدة)**  
   - Add balance: material name is a **dropdown** (from إدارة الخامات).  
   - Edit balance: only via app; each add/edit is logged in ledger with admin name.  
   - **عرض السجل:** For each material row, a "سجل" button opens a dialog with ledger entries: date/time, type (إضافة / تعديل / سحب تقرير يومي), quantity (+/-), user name.

4. **خصم تلقائي من مخزن المشروع**  
   - When a site engineer saves a daily report, the **project selected at the start of the report** is used.  
   - For each material line in the report (خامات وكميات), the app deducts the quantity from that project’s stock (matching material name and unit) and adds a ledger entry (سحب، اسم المهندس، تاريخ التقرير).

5. **إدارة التشوينات (Building materials)**  
   - "الخامة" in "إضافة تشوينات للمبنى" is a **dropdown** filled from إدارة الخامات.

6. **التقرير اليومي - الخامات (Materials step)**  
   - On narrow/mobile width: fields stack vertically to avoid overflow.  
   - Material and unit dropdowns show selected value with ellipsis; no overflow messages.

## Technical Notes

- **Database:** New table `project_stock_ledger` and migration in `database_service` (version 11). Web storage mirrors behavior via `_projectStockLedgerKey`.
- **Storage API:** Both `DatabaseService` and `WebStorageService` implement:
  - `getAttendanceForUserOnDate(userId, date)` → `(checkIn, checkOut)` for that calendar day.
  - `addProjectStockLedgerEntry(...)`, `getStockLedger(projectId, materialName)`, `deductProjectStock(...)`.
- **Reports screen:** Keeps an `_attendanceMap` keyed by `userId_year_month_day` and loads it when running the report so cards and PDF can show check-in/check-out without extra queries per card.
- **Dependencies:** `pubspec.yaml` includes pdf, printing, intl, file_picker, Firebase (firebase_core, cloud_firestore, firebase_auth). `pubspec.lock` and platform plugin registrants were regenerated.
- **Git:** Report is based on uncommitted changes (working tree) and current Git status. No commits were required to fall within the 72-hour window for this run; the report reflects the latest development state.
